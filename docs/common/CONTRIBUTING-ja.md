# TypeFlowへのコントリビューション

TypeFlowへのご関心をありがとうございます！このドキュメントでは、コントリビューションのガイドラインと手順を説明します。

[English version](../CONTRIBUTING.md)

## 行動規範

このプロジェクトに参加することで、[行動規範](CODE_OF_CONDUCT.md)に従うことに同意したものとみなされます。

## どのようにコントリビュートできますか？

### バグの報告

バグレポートを作成する前に、既存のイシューリストを確認してください。バグレポートを作成する際は、できるだけ詳細を含めてください：

- **明確で説明的なタイトル**
- **再現手順**
- **期待される動作** vs **実際の動作**
- **スクリーンショット**（該当する場合）
- **環境**（OS、ブラウザ、Node.jsバージョン）
- **追加のコンテキスト**（ログ、エラーメッセージなど）

### 機能改善の提案

機能改善の提案はGitHubのイシューとして追跡されます。機能改善の提案を作成する際は、以下を含めてください：

- **明確で説明的なタイトル**
- **提案された機能の詳細な説明**
- **ユースケース**: この機能改善がなぜ有用なのか？
- **可能な実装方法**（アイデアがある場合）

### プルリクエスト

1. **リポジトリをフォーク**
2. **`main`から機能ブランチを作成**:
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **変更を加える**
4. **コーディングスタイルに従う**（下記参照）
5. **テストを書くまたは更新する**（該当する場合）
6. **必要に応じてドキュメントを更新**
7. **変更をコミット**:
   ```bash
   git commit -m '素晴らしい機能を追加'
   ```
   明確で説明的なコミットメッセージを使用してください
8. **フォークにプッシュ**:
   ```bash
   git push origin feature/amazing-feature
   ```
9. **プルリクエストを開く**

### 開発環境のセットアップ

1. **フォークをクローン**:

   ```bash
   git clone https://github.com/your-username/typeflow.git
   cd typeflow
   ```

2. **依存関係をインストール**:

   ```bash
   npm install
   # または
   bun install
   ```

3. **開発サーバーを起動**:

   ```bash
   npm run dev
   # または
   bun dev
   ```

4. **型チェックを実行**:

   ```bash
   npm run typecheck
   # または
   bun run typecheck
   ```

5. **リンターを実行**:
   ```bash
   npm run lint
   # または
   bun run lint
   ```

### コーディングスタイル

- **TypeScript**: 既存のコードスタイルに従う
- **React**: フックを使用した関数コンポーネントを使用
- **命名**: 説明的な名前を使用、変数/関数はcamelCase、コンポーネントはPascalCase
- **フォーマット**: プロジェクトはESLintとPrettierを使用（設定されている場合）
- **コメント**: 複雑なロジックにはコメントを追加
- **i18n**: すべてのユーザー向け文字列は国際化する（`src/i18n/`を参照）

### コミットメッセージのガイドライン

- 現在形を使用（"Add feature" ではなく "Added feature"）
- 命令形を使用（"Move cursor to..." ではなく "Moves cursor to..."）
- 最初の行は72文字以内に制限
- 最初の行の後にイシューやプルリクエストを参照

例：

```
単語管理のキーボードショートカットを追加

単語追加ダイアログを開くためのCmd+K / Ctrl+Kを実装。
#123を修正
```

### テスト

- ブラウザで変更を手動でテスト
- アプリケーションが英語と日本語の両方で動作することを確認
- エッジケースとエラーシナリオをテスト
- 新機能を追加する場合は、テストを追加することを検討（テストフレームワークが設定されている場合）

### ドキュメント

- 新機能を追加したり既存の動作を変更した場合は、README.mdを更新
- CHANGELOG.mdに変更を追加
- 新しい関数/コンポーネントにJSDocコメントを追加
- 新しいユーザー向け文字列を追加した場合は、i18nファイルを更新

### 国際化（i18n）

TypeFlowは英語と日本語をサポートしています。新しい文字列を追加する際は：

1. 英語の文字列を `src/i18n/locales/en/` に追加
2. 日本語の文字列を `src/i18n/locales/ja/` に追加
3. コンポーネントで `useTranslation` フックを使用:
   ```tsx
   const { t } = useTranslation('namespace')
   ```

### プロジェクト構造

- `src/components/` - Reactコンポーネント
- `src/hooks/` - カスタムReactフック
- `src/lib/` - ユーティリティと型定義
- `src/i18n/` - 国際化ファイル
- `docs/` - ドキュメント
